Setting and getting options

Array of values

The simplest way of using the directive is passing in an array of values as options. The select is bound to a model in your controller.

Controller:

$scope.someModel = null; $scope.selectables = [1, 2, 3];

Template

someModel value: {{someModel}}

Result:

someModel value: {{someModel}}

Array of objects

Getting whole object

You can also pass in an array of objects to be used as options. In this case you can decide which fields you want to render in your option template. If nothing else is specified, the directive will set the whole object on your model

Controller:

$scope.secondModel = null; $scope.selectableNames = [ { name : "Mauro", role : "juggler"}, { name : "Silvia", role : "chef"}, { name : "Merlino", role : "little canaglia"}, ];

Template

Selected value: {{secondModel}}

Result:

Selected value: {{secondModel}}

Getting a single property

If you want to pick only a property of your object as result of selection, you can do it by using the option-property attribute.
In our previous example, let's pick only the value of the "name" property of our options. Note that you can still use all the properties of your objects in option template.

Template

Selected value: {{secondModel}}

Result:

Selected value: {{model3}}

Getting a custom value with option-getter

You can also pick up a value computed from the selected option in a custom way. This can be done using the option-getter attribute and specifying a function call to obtain the selected value from the original option. The function call must use a parameter called option as argument.

Controller:

$scope.getOpt = function(option){ return option.name + ":" + option.role; };

Template

Selected value: {{model4}}

Result:

Selected value: {{model4}}

Options can change over time

If options are passed in via options attribute, the directive will watch the options array for changes and update the list presented to the user accordingly.
